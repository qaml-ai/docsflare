# Cloudflare AI Search

Docsflare uses the current Cloudflare AI Search Worker binding. The production `wrangler.jsonc` environment binds an instance named `docsflare-docs` as `DOCS_SEARCH`.

```json
{
  "env": {
    "production": {
      "ai_search": [
        {
          "binding": "DOCS_SEARCH",
          "instance_name": "docsflare-docs"
        }
      ]
    }
  }
}
```

## Indexing

For production, deploy the Worker and trigger the binding-backed sync endpoint:

```bash
docsflare search sync --url https://example.com/docs
```

The command sends a `POST` request to `/api/search/sync`. It does not need a Cloudflare REST API token because indexing runs inside the deployed Worker through the `DOCS_SEARCH` binding.

For local development or missing bindings, the Worker falls back to a static in-memory search over built pages. That fallback is intentionally simple and exists only so the UI remains usable before Cloudflare AI Search is configured.
